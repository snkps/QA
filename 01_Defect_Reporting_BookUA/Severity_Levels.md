# Рівні Критичності Дефектів (Severity)

Визначення рівнів критичності, що використовувались для оцінки впливу дефектів на роботу системи:

* [cite_start]**Critical:** Функціональна помилка, яка блокує роботу частини або всього додатка. [cite: 729] (Наразі не виявлено).
* [cite_start]**Major:** Порушує нормальну роботу програми, але не блокує функціональність повністю. [cite: 729] Приклад: Частковий переклад інтерфейсу; [cite_start]зникає сторінка з книгами після переходу та повернення назад. [cite: 729]
* **Average:** Не дуже важлива функціональна помилка. [cite_start]Може впливати на зручність користування. [cite: 729] Приклад: Відсутні біографії авторів; [cite_start]відсутні відгуки користувачів. [cite: 729]
* [cite_start]**Minor:** Незначні візуальні помилки. [cite: 729] [cite_start]Приклад: Відсутність кнопки "Головна". [cite: 729]
* [cite_start]**Enhancement:** Пропозиції щодо покращення. [cite: 729] [cite_start]Приклад: Додати оцінку книгам. [cite: 729]
